Demo 07: Multi-Server-Orchestrierung
=====================================
Fortgeschritten | Zwei Server

WAS WIR ZEIGEN: Ein Client kann mehrere MCP-Server gleichzeitig nutzen.
Hier: Katalog (Plaene) + Executor (Ausfuehrung)

ARCHITEKTUR:
┌────────────┐     ┌─────────────────┐
│  CLIENT    │────►│  Plan-Server    │  (Plaene lesen) = Server 06
│            │     └─────────────────┘
│            │     ┌─────────────────┐
│            │────►│  Exec-Server    │  (Plaene ausfuehren) = Server 07
└────────────┘     └─────────────────┘

VERFUEGBARE MCP-BAUSTEINE:
- Plan-Server (5000): Resources tests/catalog, tests/plan/{slug}
- Executor-Server (5001): Tools tests.list, tests.run

VORBEREITUNG (ZWEI TERMINALS + CLIENT)
--------------------------------------
Terminal 1: cd 06/server && dotnet run  (Port 5000 - Plan-Server)
Terminal 2: cd 07/server && dotnet run  (Port 5001 - Executor)
Terminal 3: cd 07/client && dotnet run

KEIN LLM NOETIG.

SCHRITT-FUER-SCHRITT (MENUE-BASIERT)
------------------------------------

1. PLAENE LISTEN (Plan-Server 06)
   Taste: 3 (Plaene listen)

   -> Zeige: Client ruft Plan-Server auf
   -> Erklaere: Erster MCP-Server im Verbund

2. PLAN LESEN
   Taste: 4 (Plan lesen)
   Eingabe: google-news

   -> Zeige: Resource tests/plan/google-news von Server 06
   -> Erklaere: Plan ist nur Beschreibung

3. PLAN AUSFUEHREN (Exec-Server 07)
   Taste: 5 (Plan ausfuehren)

   -> Zeige: Tool tests.run auf Executor-Server
   -> Erklaere: Zweiter Server fuehrt tatsaechlich aus!
   -> Zeige: Ergebnis der HTTP-Requests

4. EXECUTOR-TOOLS ANZEIGEN
   Taste: 6 (Executor-Tools listen)

   -> Zeige: tools/list auf Executor-Server
   -> Erklaere: Jeder Server hat eigene Tools

FLOW ZUSAMMENFASSUNG
--------------------
1. Katalog: Testplan nachschlagen (Server 06)
2. Executor: Plan ausfuehren, Ergebnis pruefen (Server 07)

KERNAUSSAGE
-----------
"Ein Client, mehrere Server.
 Jeder Server hat seine Verantwortung.
 Das ist die Skalierung mit MCP."
