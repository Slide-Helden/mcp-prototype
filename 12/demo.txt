Demo 12: MCP ohne KI
====================
Sonderfall | MCP als Standard-Connector | Dauer: ~5 Min

LERNZIEL: MCP funktioniert auch komplett ohne LLM.
Das Protokoll ist ein reiner Connector-Standard.

KERNAUSSAGE:
MCP ist NICHT nur fuer KI. Es ist ein standardisiertes Protokoll
fuer Tool/Resource-Zugriff - nutzbar in jedem Szenario.

USE-CASE: Ops-Dashboard fuer .NET Services

VERFUEGBARE MCP-BAUSTEINE:
- Tools: ops.services.list, ops.service.restart, ops.service.deploy, ops.service.maintenance, ops.timeline.note
- Resources: ops/overview, ops/service/{id}, ops/timeline, ops/runbook/{topic}
- Services: api, worker, billing, web, tests
- Runbooks: restart-service, fix-nullable-storm, nuget-cache-panic, tests-red-green, deploy-blue-green, incident-first-response

VORBEREITUNG
------------
Terminal 1: cd 12/server && dotnet run
Terminal 2: cd 12/client && dotnet run

Server laeuft auf http://localhost:5600/sse
KEIN LLM NOETIG. Kein API-Key erforderlich.

SCHRITT-FUER-SCHRITT (MENUE-BASIERT)
------------------------------------

1. STATUS-UEBERSICHT
   Taste: 1 (Status-Uebersicht lesen)

   -> Zeige: Resource ops/overview
   -> Erklaere: Reiner MCP-Aufruf, kein LLM

2. SERVICES AUFLISTEN
   Taste: 2 (Services listen)

   -> Zeige: Tool ops.services.list gibt Tabelle zurueck
   -> Erklaere: Tool-Call ohne LLM - der Client entscheidet

3. SERVICE-DETAILS
   Taste: 3 (Service-Details lesen)
   Eingabe: web

   -> Zeige: Resource ops/service/web
   -> Erklaere: Dynamische URI mit Parameter

4. SERVICE NEU STARTEN
   Taste: 4 (Service neu starten)
   Eingabe: tests

   -> Zeige: Tool ops.service.restart wird aufgerufen
   -> Erklaere: MCP kann auch Aktionen ausloesen, nicht nur lesen

5. DEPLOYMENT MARKIEREN
   Taste: 5 (Deployment markieren)
   Eingabe: web, Version: 9.0.0, Notiz: Hotfix

   -> Zeige: Tool ops.service.deploy
   -> Erklaere: Strukturierte Parameter

6. TIMELINE ANZEIGEN
   Taste: 7 (Timeline anzeigen)

   -> Zeige: Alle manuellen Aktionen protokolliert
   -> Erklaere: Resource als Audit-Log

7. RUNBOOK LESEN (OPTIONAL)
   Taste: 8 (Runbook anzeigen)
   Eingabe: fix-nullable-storm

   -> Zeige: Runbook mit C#-spezifischen Anweisungen
   -> Erklaere: Resources koennen auch Dokumentation sein

KERNAUSSAGE
-----------
"MCP ist ein Protokoll - nicht mehr, nicht weniger.
 Ob ein LLM die Aufrufe macht oder ein Skript: egal.
 Das macht MCP universell einsetzbar."
