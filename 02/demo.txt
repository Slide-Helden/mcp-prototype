Demo 02: LLM-first mit Dokumenten
==================================
Aufbauend auf Demo 01 | +Suche, +Quellen

WAS WIR ZEIGEN: Das LLM durchsucht einen Dokumentenkatalog, liest Inhalte
und zitiert Quellen - alles autonom.

VERFUEGBARE MCP-BAUSTEINE:
- Tools: docs.search (keyword), docs.random
- Resources: docs/catalog, docs/document/{id}, docs/summary/{id}
- Dokumente: csharp-linq-love, csharp-see-sharp, csharp-async-cake,
             java-license-latte, java-oracle-fee, standup, coffee-break, etc.

VORBEREITUNG
------------
Terminal 1: cd 02/server && dotnet run
Terminal 2: cd 02/client && dotnet run

Server laeuft auf http://localhost:5000/sse

SCHRITT-FUER-SCHRITT
--------------------

1. EINSTIEG - Katalog lesen
   Eingabe: Lies den Dokumentenkatalog und zeig mir was verfuegbar ist.

   -> Beobachte: LLM ruft mcp.read_resource mit "docs/catalog" auf
   -> Erwartet: Markdown-Liste mit Dokumenten (IDs, Titel, Tags)

2. KATALOG ERKUNDEN
   Eingabe: Zeig mir alle verfuegbaren Dokumente.

   -> Beobachte: LLM liest docs/catalog
   -> Erwartet: Markdown-Liste mit Titeln, IDs und Tags

3. GEZIELTE SUCHE
   Eingabe: Suche nach Dokumenten ueber C#.

   -> Beobachte: LLM ruft docs.search mit keyword "csharp" auf
   -> Erwartet: Liste mit csharp-linq-love, csharp-see-sharp, etc.

4. DOKUMENT MIT QUELLE LESEN
   Eingabe: Lies das Dokument csharp-linq-love und fasse es zusammen.

   -> Beobachte: LLM ruft mcp.read_resource mit "docs/document/csharp-linq-love" auf
   -> Erwartet: Volltext des Dokuments + Zusammenfassung

5. VERGLEICH MEHRERER DOKUMENTE
   Eingabe: Vergleiche csharp-linq-love und csharp-async-cake. Was haben sie gemeinsam?

   -> Beobachte: LLM liest beide Dokumente nacheinander
   -> Erwartet: Vergleich mit Quellenangaben

6. ZUFALLSDOKUMENT
   Eingabe: Gib mir ein zufaelliges Dokument.

   -> Beobachte: LLM ruft docs.random auf
   -> Erwartet: Zufaellige Empfehlung mit ID und Zusammenfassung

7. JAVA-SUCHE (OPTIONAL)
   Eingabe: Was sagen die Dokumente ueber Java?

   -> Beobachte: LLM sucht mit keyword "java"
   -> Erwartet: java-license-latte, java-oracle-fee, etc.

UEBERLEITUNG ZU DEMO 03
-----------------------
"Bisher hat das LLM alles selbst entschieden.
 Aber was, wenn WIR jeden Schritt kontrollieren wollen?
 Das ist der Orchestrator-first Ansatz."

NOTFALL-BEFEHLE
---------------
:tools              - Zeigt docs.search, docs.random
:resources          - Zeigt docs/catalog, docs/document/{id}, docs/summary/{id}
:read docs/catalog  - Katalog manuell lesen
:exit               - Beenden
